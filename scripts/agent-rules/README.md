# Cross-platform agent rules

One manifest, every IDE. The *content* an agent needs (source-media safety,
per-domain routing between the live and offline servers) is platform-neutral and
lives in `AGENTS.md`, `docs/kernels/*`, `docs/guides/*`, and
`resolve-advanced/README.md`. What differs per IDE is only the **trigger/discovery
layer** — the file each tool reads and how it decides *when* to load it. This
directory generates those thin per-platform shims from a single source so they
cannot drift.

## Two layers

1. **Dynamic, every-MCP-client layer — MCP prompts (not generated here).** The
   server registers per-domain slash prompts in `src/server.py`
   (`color_grade_workflow`, `timeline_edit_workflow`, `conform_workflow`,
   `delivery_workflow`, plus `analyze_media`). Every MCP client — Codex, Cursor,
   Copilot, Continue, Claude Desktop, Cline, Roo, Zed, Windsurf, OpenCode — gets
   them the moment it connects, with no committed files. This is the only layer
   Claude Desktop (a chat client with no repo-rule mechanism) can use.

2. **Static per-repo-file layer — generated by `generate.mjs`.** For always-on
   rules and glob/semantic-triggered loading, each IDE reads a different file:

   | Platform | File(s) | Trigger | Source |
   |---|---|---|---|
   | Codex / OpenCode / Zed / others | `AGENTS.md` (`## Domain Routing` block) | always-on | generated block |
   | Claude Code | `.claude/skills/*.md` | semantic (description) | hand-authored (rich) |
   | Cursor | `.cursor/rules/*.mdc` | `alwaysApply` repo rule + per-domain `description` (agent-requested) | generated |
   | VS Code / Copilot | `.github/copilot-instructions.md` + `.github/instructions/*.instructions.md` | always-on + `applyTo` | generated |
   | Windsurf | `.windsurf/rules/*.md` + `.windsurfrules` | rules dir + legacy flat | generated |
   | Cline | `.clinerules` (single file) | always-on | generated |
   | Roo Code | `.roo/rules/resolve-mcp.md` | always-on | generated |
   | Continue | `.continue/rules/resolve-mcp.md` | always-on | generated |
   | Claude Desktop | — (chat client, no repo rules) | — | MCP prompts only |

   `.claude/skills/*` stays hand-authored (Claude's rich semantic skills); every
   other file is generated. `AGENTS.md` remains the universal backstop for any
   client not listed. Always-on rule dirs (Cline/Roo/Continue) get one compact
   combined file (repo hygiene + domain table), since they load every rule file
   unconditionally.

## Drift-proofing

- Tool/action counts are **parsed from their canonical docs** at generate time,
  never hand-copied: compound/granular from `docs/SKILL.md`, kernel actions from
  `docs/kernels/README.md`, advanced tools from `resolve-advanced/README.md`.
- `tests/test_agent_rules_drift.py` runs `generate.mjs --check` and fails the
  suite if any generated file is stale, and asserts the four domain prompts exist
  in `src/server.py`.
- `tests/test_doc_tool_counts.py` independently asserts the canonical docs quote
  the *real* code counts. Chain: code → `test_doc_tool_counts` → canonical docs →
  `generate.mjs` → agent-rule files → `test_agent_rules_drift`.

## Usage

```bash
node scripts/agent-rules/generate.mjs          # write/update all outputs
node scripts/agent-rules/generate.mjs --check   # exit 1 if anything is stale (CI/release gate)
```

## Adding a domain or a platform

- **New domain:** add an entry to `DOMAINS` in `generate.mjs`, add a matching
  `@mcp.prompt` in `src/server.py`, and (optionally) a rich `.claude/skills/*.md`.
  Regenerate.
- **New platform:** add an `emit(...)` for its convention in `generate.mjs`,
  reusing `domainBody(d)` / `repoHygiene`. Regenerate. Do not hand-edit generated
  files — the drift guard will fail.
